# Weather API Service

Асинхронный HTTP-сервер для предоставления информации о погоде с использованием Open-Meteo API.

Реализован в рамках тестового задания на позицию Python-разработчика (стажёр) и переработан в микросервис для портфолио.

---

## Используемые технологии

- Python 3.10+
- FastAPI
- SQLite (aiosqlite)
- httpx
- pytest + pytest-asyncio
- Docker

---

## Запуск сервера локально

1. Установить зависимости:
```bash
pip install -r requirements.txt
```

2. Запустить сервер из корня проекта:
```bash
uvicorn app.main:app --reload
```

После запуска сервер будет доступен по адресу:
```
http://127.0.0.1:8000
```

Также доступна Swagger-документация:
```
http://127.0.0.1:8000/docs
```

---

## Запуск через Docker

Для запуска проекта в контейнере Docker:

1. Собрать образ:
```bash
docker build -t async-weather-api .
```

2. Запустить контейнер:
```bash
docker run -d -p 8000:8000 async-weather-api
```

## API методы

### 1. Регистрация пользователя

**POST** `/register`

Регистрирует нового пользователя.

**Тело запроса:**
```json
{
  "username": "alice"
}
```

**Ответ:**
```json
{
  "username": "alice",
  "id": 1
}
```

---

### 2. Получение текущей погоды по координатам

**GET** `/weather/current`

**Параметры запроса:**
- `latitude` — широта
- `longitude` — долгота

**Пример запроса:**
```
/weather/current?latitude=55.75&longitude=37.61
```

**Ответ:**
```json
{
  "temperature": 22.1,
  "wind_speed": 5.4,
  "pressure": 1013.2
}
```

---

### 3. Добавление города пользователю

**POST** `/cities`

Добавляет город в список пользователя и сразу загружает прогноз погоды.

**Тело запроса:**
```json
{
  "user_id": 1,
  "name": "Moscow",
  "latitude": 55.75,
  "longitude": 37.61
}
```

**Ответ:**
```json
{
  "user_id": 1,
  "name": "Moscow",
  "latitude": 55.75,
  "longitude": 37.61,
  "id": 1
}
```

> Для одного пользователя нельзя добавить один и тот же город дважды.

---

### 4. Получение списка городов пользователя

**GET** `/cities`

**Параметры запроса:**
- `user_id` — ID пользователя

**Пример запроса:**
```
/cities?user_id=1
```

**Ответ:**
```json
[
  {
    "id": 1,
    "user_id": 1,
    "name": "Moscow",
    "latitude": 55.75,
    "longitude": 37.61
  }
]
```

---

### 5. Получение прогноза погоды на конкретное время

**GET** `/weather/forecast`

Возвращает прогноз погоды для города на текущий день в указанное время.

**Параметры запроса:**
- `user_id` — ID пользователя
- `city_name` — название города
- `time` — время в формате ISO 8601 (например, `2026-01-01T14:00`; требуется почасовой формат времени `XX:00`)
- `params` (опционально) — список параметров погоды:
  - `temperature`
  - `humidity`
  - `wind_speed`
  - `precipitation`
  - `pressure`

**Пример запроса:**
```
/weather/forecast?user_id=1&city_name=Moscow&time=2026-01-01T14:00&params=temperature&params=wind_speed
```

**Ответ:**
```json
{
  "temperature": -4.4,
  "wind_speed": 6.6
}
```

Если `params` не указаны, возвращаются все доступные параметры.

---

## Обновление прогноза

- Прогноз погоды обновляется каждые **15 минут**
- Обновление выполняется в фоновом асинхронном процессе
- Используется почасовой прогноз Open-Meteo на текущий день
